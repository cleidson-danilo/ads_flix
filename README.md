## 👥 Organização do Grupo e Critérios de Avaliação

[span_6](start_span)Conforme os requisitos estabelecidos pelo professor, cada integrante ficou responsável por uma parte técnica específica do ecossistema do app, garantindo que todos dominem o funcionamento na apresentação[span_6](end_span):

### 🧠 Parte 1: O "Coração" dos Dados (Back-End / Service)
* **[span_7](start_span)[span_8](start_span)📡 Integrante 1:** Configuração global e injeção do `HttpClient` no projeto (`app.config.ts`)[span_7](end_span)[span_8](end_span).
* **[span_9](start_span)🛜 Integrante 2 (@Nascimento🔰):** Criação do Serviço (`services/cinema`) e métodos **GET** para chamadas da API[span_9](end_span).

### 📺 Parte 2: Interfaces e Controle de Fluxo (Front-End)
* **[span_10](start_span)[span_11](start_span)🎬 Integrante 3:** Construção da Tela Inicial (Home) utilizando a diretiva estrutural `&#64;for` para listagem dinâmica de filmes populares[span_10](end_span)[span_11](end_span).
* **[span_12](start_span)[span_13](start_span)[span_14](start_span)🔍 Integrante 4:** Criação da Segunda Tela (Buscar) com barra de pesquisa para localização de títulos e lógica condicional `&#64;if`[span_12](end_span)[span_13](end_span)[span_14](end_span).

### 🎨 Parte 3: Customização e Tratamento de Dados (Pipes e Diretivas)
* **[span_15](start_span)🎭 Integrante 5:** Implementação de Pipes no projeto (Pipe Built-in para formatar dados e Pipe Customizado para limitar o tamanho da sinopse)[span_15](end_span).
* **[span_16](start_span)✨ Integrante 6:** Desenvolvimento da Diretiva Personalizada (`appAmpliar`) usando `&#64;HostListener` e `&#64;HostBinding` para efeitos visuais nos cards[span_16](end_span).

### 🗺️ Parte 4: Navegação e Bônus
* **[span_17](start_span)[span_18](start_span)🚀 Integrante 7:** Configuração de rotas dinâmicas e passagem de parâmetros por rota para a Tela de Detalhes, garantindo a pontuação extra[span_17](end_span)[span_18](end_span).
